# Customer Purchase Value Prediction

An end-to-end machine learning project developed for the **Machine Learning Project course in the IIT Madras Diploma in Data Science** and evaluated through a Kaggle competition.

The objective was to predict the purchase value generated by an e-commerce session using anonymized behavioral, acquisition, device, and geographic information.

## Business problem

Digital commerce platforms collect large amounts of session-level information, but most sessions do not immediately produce a purchase. Estimating the expected purchase value of each session can support:

- customer and session prioritization;
- marketing-budget allocation;
- channel and campaign evaluation;
- personalized engagement;
- identification of high-potential traffic.

The competition used **R²** as its evaluation metric.

## Dataset

Each row represents one user session.

- **Training observations:** 116,023
- **Test observations:** 29,006
- **Raw predictors:** 51
- **Target:** `purchaseValue`

Feature groups include:

- engagement metrics such as page views, total hits, bounces, and session number;
- device and browser information;
- traffic source, medium, campaign, and referral attributes;
- country, continent, city, region, and other geographic indicators;
- session date and timestamp information.

The competition data are not included in this repository because they are subject to the competition rules.

## Technical workflow

1. Audited missingness, cardinality, constant columns, and the highly zero-inflated target.
2. Converted raw date and Unix timestamp fields into interpretable calendar features.
3. Replaced placeholder strings with missing values and removed non-informative identifiers and constant columns.
4. Built reusable preprocessing pipelines for numerical, low-cardinality categorical, and high-cardinality categorical features.
5. Compared baseline and tree-based regression approaches.
6. Used XGBoost with fold-safe preprocessing and hyperparameter tuning.
7. Evaluated models on an untouched holdout set using R², MAE, and RMSE.
8. Generated non-negative predictions in the required Kaggle submission format.

## Validation design

The portfolio notebook uses a **split-first** workflow:

- the holdout set is created before target-aware preprocessing or weighting;
- target encoding is fitted only on training data;
- any emphasis placed on non-zero purchases is implemented through training weights rather than duplicating validation observations;
- all reported holdout metrics are calculated on the original, untouched target distribution.

This avoids optimistic validation estimates in a zero-inflated regression problem.

## Running the notebook

On Kaggle, attach the competition dataset and run the notebook. It automatically checks:

```text
/kaggle/input/engage-2-value-from-clicks-to-conversions/
```

For local execution, place the three competition files in either `data/` or `../data/`:

- `train_data.csv`
- `test_data.csv`
- `sample_submission.csv`

Install dependencies with:

```bash
pip install -r requirements.txt
```

## Results

The original experimentation notebooks record a best Kaggle score of **0.55745**. Confirm the exact public/private leaderboard score and final rank in Kaggle before adding either number to a CV or public profile.

The cleaned notebook intentionally does not hard-code unverified results. Run all cells and update this section with:

- holdout R²;
- Kaggle public/private R²;
- final competition rank or percentile;
- strongest validated model configuration.
